# リスク評価フレームワーク

このフレームワークは、AI生成出力、コード変更、自動化、意思決定支援の提案が、システムリスクを高めるかどうかを評価するためのものです。

公開テスト、AIコードエージェント、人間のレビューに使うことを想定しています。

---

## リスクレベル

| レベル | 意味 | 必要な対応 |
|---|---|---|
| 低 | 局所的、可逆的、透明、影響範囲が限定的 | 通常の検証で進める |
| 中 | 外部影響、依存性、失敗モードの不明確さがある | リスクを説明し、安全策を追加する |
| 高 | 不可逆、拡張可能、不透明、安全上重要な影響がある | 明示的な人間レビューを必要とする |
| 重大 | 害、強制、欺瞞、違法行為、システム崩壊リスクを可能にする | 拒否する、または安全に再設計する |

---

## リスク次元

| 次元 | 懸念例 |
|---|---|
| 規模 | 多くの人やシステムへ急速に影響するか |
| 不可逆性 | 停止後も害が残るか |
| 不透明性 | 人間が結果を理解または監査できないか |
| 自律性 | 人間確認なしに行動するか |
| データ感度 | 個人情報、金融、医療、認証情報へアクセスするか |
| 外部行動 | 送信、削除、公開、デプロイ、取引、実リソース変更を行うか |
| 生態影響 | 採掘、廃棄、エネルギー負荷、生態系損傷を増やすか |
| 社会影響 | 強制、操作、不信、排除を増やすか |
| セキュリティ影響 | 認証、ログ、権限、監視を弱めるか |

---

## 人工叡智の判断ルール

同じ目的を、より低いリスク、より高い透明性、より良い可逆性で達成できるなら、安全な道を選ぶべきです。

ユーザーが有害な実装を求めた場合は、正当な目的を保持しつつ、方法を安全に再設計します。
