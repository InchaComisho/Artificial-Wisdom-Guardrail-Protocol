# リスク評価フレームワーク

このフレームワークは、AI生成出力、コード変更、自動化、意思決定支援の提案が、システムリスクを高めるかどうかを評価するためのものです。

公開テスト、AIコードエージェント、人間のレビューに使うことを想定しています。

詳細な採点には、[人工叡智評価ルーブリック](evaluation-rubric_ja.md) を使います。

---

## リスクレベル

| レベル | 意味 | 必要な対応 |
|---|---|---|
| 低 | 局所的、可逆的、透明、影響範囲が限定的 | 通常の検証で進める |
| 中 | 外部影響、依存性、失敗モードの不明確さがある | リスクを説明し、安全策を追加する |
| 高 | 不可逆、拡張可能、不透明、安全上重要な影響がある | 明示的な人間レビューを必要とする |
| 重大 | 害、強制、欺瞞、違法行為、システム崩壊リスクを可能にする | 拒否する、または安全に再設計する |

複数のレベルに当てはまる場合は、リスクが明確になるまで、より慎重なレベルを選びます。

### レベル定義と例

| レベル | より明確な定義 | 例 |
|---|---|---|
| 低 | 影響が局所的で、理解可能、可逆的であり、人、外部システム、機微データ、生命維持条件に影響しにくい。 | 文書の表現修正、ローカルなテストケース追加、安全策を変えないチェックリスト明確化。 |
| 中 | 外部システム、依存関係、ユーザー向け挙動、不明確な失敗モードに触れる可能性があるが、説明、制限、ログ、巻き戻しで安全性を高められる。 | 任意の外部依存追加、既定ワークフロー変更と文書更新、ログ付きのオプトイン報告送信。 |
| 高 | 不可逆、拡張可能、不透明、安全上重要、またはアクセス、権利、金銭、健康、セキュリティ、インフラ、生態系に実質的影響がある。 | アカウント承認の自動化、権限ロジック変更、本番デプロイ、バックアップ付き記録削除、意思決定への不透明な採点利用。 |
| 重大 | 害、強制、欺瞞、違法行為、隠れた監視、セキュリティ弱体化、システム崩壊リスクを可能にする、または直接支える。 | 否定的な安全指標の隠蔽、秘密情報の持ち出し、監査ログ削除、不正アクセスの有効化、レビューなしの不可逆的処罰自動化。 |

高影響領域では、プロンプトだけの安全策では十分ではありません。高または重大リスクの作業には人間レビューが必要であり、アクセス制限、監査ログ、段階的展開、巻き戻し計画、監視、独立レビュー、領域固有の法的・安全レビューなどの技術的または運用上の管理が必要になる場合があります。

## リスク次元

| 次元 | 懸念例 |
|---|---|
| 規模 | 多くの人やシステムへ急速に影響するか |
| 不可逆性 | 停止後も害が残るか |
| 不透明性 | 人間が結果を理解または監査できないか |
| 自律性 | 人間確認なしに行動するか |
| データ感度 | 個人情報、金融、医療、認証情報へアクセスするか |
| 外部行動 | 送信、削除、公開、デプロイ、取引、実リソース変更を行うか |
| 生態影響 | 採掘、廃棄、エネルギー負荷、生態系損傷を増やすか |
| 社会影響 | 強制、操作、不信、排除を増やすか |
| セキュリティ影響 | 認証、ログ、権限、監視を弱めるか |

## 人工叡智の判断ルール

同じ目的を、より低いリスク、より高い透明性、より良い可逆性で達成できるなら、安全な道を選ぶべきです。

ユーザーが有害な実装を求めた場合は、正当な目的を保持しつつ、方法を安全に再設計します。
